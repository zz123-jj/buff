# 能量机关识别模块

## 1.目标：

接收每一帧后触发回调得到：
    
1. 相机拍摄此帧的时间戳
2. 能量机关目前模式（大或小）
3. 旋转方向（顺 逆）
4. 目标扇页Box中心位置（像素坐标系）
5. 中心RBox中心位置 （像素坐标系）
6. 能量机关半径 （用于后续解算深度）

触发‘正在跟随时’发送以上信息

- [ ] 旋转方向 只测过人眼视角的逆时帧

# 能量机关数据解算模块

## 1.目标：

收到来自识别模块的消息后进行解算：

1. 传递此帧拍摄时间戳
2. 传递能量机关模式
3. 传递旋转方向
4. 小能量机关模式下只获取单个扇页odom系下的三维坐标
5. 大能量机关模式下 解算角速度三角函数参数及开始时间（数据拟合需要时间，在没得到参数时默认0） 同时获取 两片扇页（要不要呢）odom系下的三维坐标

## 2.目前存在问题已解决：

- [x] is_tracking = false时 重置所有消息为0 is_buff_mode = true = 只重置参数消息为0
- [ ] 时间戳不知道有没有对齐
- [ ] 拟合准确性有待考量
- [ ] 解算的odom坐标不知道准不准
- [ ] 代码臃肿
## 3.回调链路：

接收到来自detector的消息，按大小能量机关分为两路，
小能量机关模式下只负责转换坐标
大能量机关模式下要收集数据点转换为角度后做一次拟合（只做固定窗口长度的拟合）为减少算力开销只做一次

传递时间戳 -> 传递能量机关模式 -> 传递旋转方向 -> 解算扇页坐标 -> 判断is_big_buff -> true就转换角度 到达窗口长度后 做一次拟合 -> 发送消息
